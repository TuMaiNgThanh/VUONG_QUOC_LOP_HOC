# Security Policy

## Supported Versions

The actively maintained branch is `main`.

## Reporting a Vulnerability

If you discover a security issue, please do not create a public issue immediately.

Please report privately with:

- Affected file(s) and module(s)
- Reproduction steps
- Impact assessment
- Suggested fix (if available)

Temporary contact placeholder:

- security@your-domain.com

## Response Target

- Initial acknowledgment: within 72 hours
- Triage decision: within 7 days
- Fix target: depends on severity

## Severity Guidance

- Critical: auth bypass, data exposure, remote code execution
- High: privilege escalation, stored XSS, rules misconfiguration
- Medium: non-critical injection vectors, weak validation
- Low: hardening and best-practice improvements
